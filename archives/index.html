vc6下编译curl

今天，天气有雨。温度9到2度。结束春节长假，见了一些同学，也应该把心静下来工作了。

重新把年前的工作整理一下，还有一个遗留问题就是如何把代码移植到VC6.0下（有时候项目就是这样，用什么工具由不了你）

首先，用vitrualbox安装了xp系统，vc6.0在虚拟机下跑起来很快，这使得我有点想返朴归真的感觉，也理解了老姚为啥要用vc6开发。

在编译curl时还是遇到原来的问题。这里只把成功步骤简略：

  1. 下载openssl，切记下载openssl-0.9.8g, 因为新的版本不支持vc6.0，编译步骤如下：

    • 首先要安装 ActivePerl
    • 命令行方式进入openssl目录
    • perl Configure VC-WIN32 —prefix=C:\openssl
    • ms\do_ms
    • 编译静态库 nmake -f ms\nt.mak 【注意，可以修改nt.mat的代码生成方式为MT或MTd，默认为MD】
    • nmake -f ms\nt.mak install

  2. 下载zlib，编译步骤如下：

    • 命令行方式进入zlib目录
    • nmake -f win32/Makefile.msc 【注意，可以修改代码生成方式为MT或MTd，默认为MD】

  3. 最后编译libcurl， 在curl-XXX\lib目录下新建build.bat,输入以下内容

    call “C:/Program Files/Microsoft Visual Studio/VC98/Bin/vcvars32.bat”
    set CFG=release-ssl-zlib 【这里可以修改为debug】
    set OPENSSL_PATH=C:/Libs/openssl-0.9.8g
    set ZLIB_PATH=C:/Libs/zlib-1.2.8
    set WINDOWS_SSPI=0
    set RTLIBCFG=static 【这里可以指明是否生成静态库】
    nmake -f Makefile.vc6 【注意，可以修改代码生成方式为MT或MTd，默认为MD】

  4. 把curl/include及/lib/libcurl.lib, zlib.lib, libeay32.lib, ssleay32.lib拷贝出来即可以使用。

打包给老姚，搭建完博客，已是1点。这是2014年的第9天。
